3. 渲染数据

在我们的store.js中添加一些假数据

todos: [
  {id: Math.random() * 10, title: "吃饭", done: false},
  {id: Math.random() * 10, title: "睡觉", done: false},
  {id: Math.random() * 10, title: "打游戏", done: false},

然后渲染列表数据,类completed表示的是完成时的类,当每一项的done为true时,才添加这个类

<li :class="{'completed': item.done}" v-for="item in $store.state.todos" :key="item.id">
    <div class="view">

4. 添加todo

在store.js中定义一个mutations,添加一个addTodo的方法

mutations: {
    addTodo(state) {
      state.todos.push({id: Math.random() * 10, title: title, done: false});
    }
}

然后在App.vue中methods创建对应的方法,并且我们再这里过滤一下,如果用户输入为空的话,取消输入

addTodo(e) {
  let title = e.target.value;
  if(title.trim().length == 0){
      return;
  }
  this.$store.commit("addTodo", title);
  // 清空文本框
  e.target.value = "";
}

最后给元素绑定事件 $event可以获取元素的属性 ,添加事件修饰符,按住enter时才添加

<input class="new-todo" @keyup.enter="addTodo($event)" placeholder="What needs to be done?" autofocus>

5. 删除todo

一样的套路

在store.js中的mutations中定义一个方法,deleteTodo

deleteTodo(state, id) {
  state.todos.forEach((t, i) => {
    if(t.id == id){
      state.todos.splice(i, 1);
      return;
    }
  })
}

在App.vue中定义对应的方法,deleteTodo

deleteTodo(id){
    this.$store.commit("deleteTodo", id);
}

给对应的删除按钮添加事件,传参

<button class="destroy" @click="deleteTodo(item.id)"></button>

6. 修改状态

还是一样

在store.js中的mutations中定义一个方法,changeTodo

changeTodo(state, {id, e}) {
  state.todos.forEach(t => {
    if(t.id == id){
      t.done = e.target.checked;
      return;
    }
  })
}

在App.vue中定义一个对应的方法,传参

changeTodo(id, e){
    this.$store.commit("changeTodo", {id, e});
}

给对应的按钮添加事件

<input class="toggle" type="checkbox" :checked="item.done" @change="changeTodo(item.id, $event)">

保存

在失去焦点或者按下enter时保存

在store.js中定义一个方法,saveEdit

saveEdit(state, {e, id}){
  let title = e.target.value;
  state.todos.forEach(t => {
    if(t.id == id){
      t.title = title;
      state.editing = {};
      return;
    }
  })
}

在App.vue中定义一个对应的方法,saveEdit

saveEdit(e, id){
    this.$store.commit("saveEdit", {e, id});
}

给输入框绑定事件

<input class="edit" :value="item.title" @keyup.enter="saveEdit($event, item.id)" @blur="saveEdit($event, item.id)">

这样我们的按下enter和失去焦点保存就做好了

取消保存

当用户不想保存的时候,可以按下esc取消

在store.js中定义一个cancelSave,顾名思义:取消保存的意思,

cancelSave(state, e){
  // 有的地方会有一些bug
  e.target.value = state.editing.title;
  state.editing = {};
}

在App.vue中定义对应的方法cancelSave

cancelSave(e){
    this.$store.commit("cancelSave", e);
}

给输入框添加keyup的esc取消事件,esc的keyCode值是27

<input class="edit" :value="item.title" @keyup.enter="saveEdit($event, item.id)" @blur="saveEdit($event, item.id)" @keyup.27="cancelSave($event)">

8. 全选

点击输入框左边的下箭头时,实现全选,和取消全选,(全选是亮起,未全选是暗)

在store.js中的getters中定义一个方法isAllCheck,这个方法用来监听todo.done的状态即是否全部都是true

在这里我们用一个方法every

every() 方法用于检测数组所有元素是否都符合指定条件（通过函数提供）。

every() 方法使用指定函数检测数组中的所有元素：

  • 如果数组中检测到有一个元素不满足，则整个表达式返回 false ，且剩余的元素不会再进行检测。
  • 如果所有元素都满足条件，则返回 true。
isAllCheck(state) {
  return state.todos.every(t => t.done);
}

然后我们在App.vue中使用这个getters,用来监听todos的状态

<input id="toggle-all" class="toggle-all" type="checkbox" :checked="$store.getters.isAllCheck">

11. 切换列表

点击all查看所有,点击active查看todos未完成的项,点击completed查看已经完成的项

首先,我们再store.js中的state中定义一个属性visibility,下面的getters根据这个属性来返回对应的列表

在store.js中的getters定义一个getter,名为getTodoList

getTodoList(state) {
  if(state.visibility == "all"){
    return state.todos;
  } else if (state.visibility == "active"){
    return state.todos.filter(t => !t.done);
  } else if (state.visibility == "completed"){
    return state.todos.filter(t => t.done);
  }
}

在App.vue中替换一下li渲染的列表,

<li :class="{'completed': item.done, 'editing': $store.state.editing == item}" v-for="item in $store.getters.getTodoList" :key="item.id">

现在我们可以来看一下,点击all,active,completed时候地址栏的变化,刷新的时候地址栏是不会变的,我们根据这个特性来设置列表

补充浏览器对象window有一个事件叫做onhashchange,用它可以来监听锚点的变化

onhashchange 事件在当前 URL 的锚部分(以 '#' 号为开始) 发生改变时触发 。

锚部分的实例：指定当前 URL 为

http://www.example.com/test.htm#part2 - 这个 URL 中的锚部分为 #part2。

你可以使用以下方式调用事件:

  • 通过设置Location 对象 的 location.hash 或 location.href 属性修改锚部分。
  • 使用不同书签导航到当前页面（使用"后退" 或"前进"按钮）
  • 点击链接跳转到书签锚

现在我们回到store.js中定义一个方法,用来修改visibility,方法名为visibilityChange

visibilityChange(state, val) {
  state.visibility = val;
}

然后我们在App.vue中定义一个方法用来监听hash的变化: hashChange

hashChange() {
      const hash = window.location.hash.substr(1);
      if(hash == "/"){
        this.$store.commit("visibilityChange", "all");
      } else if (hash == "/active"){
        this.$store.commit("visibilityChange", "active");
    } else if (hash == "/completed"){
        this.$store.commit("visibilityChange", "completed");
    }
}

然后在我们的vue的生命周期钩子created调用事件

created() {
    this.hashChange();
    window.onhashchange = this.hashChange;
}

12. 本地化存储

在数据发生变化的时候,修改本地中的数据

我们可以使用 JSON.stringify() 方法将 JavaScript 对象转换为字符串。

我们可以使用 JSON.parse() 方法将数据转换为 JavaScript 对象。

在store.js中把假数据修改为获取本地的数据,如果为空的话,将todos赋值为空数组

todos: JSON.parse(window.localStorage.getItem("todos") || "[]"),

回到App.vue中添加一个生命周期钩子,用来修改本地数据

updated() {
    window.localStorage.setItem("todos", JSON.stringify(this.$store.state.todos));
}
